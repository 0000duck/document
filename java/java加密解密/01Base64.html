描述
由于历史原因e-mail只能传输ascill码文件
Base64用于把事件加密为固定的64个ascill字符(故叫Base64)以解决此问题
sun默认实现了此算法但其未完全安照标准 故不推荐用 可选择BouncyCastle的实现

Base64
UrlBase64

Base64
org.bouncycastle.util.encoders.Base64

System.out.println("this is Base64");
String de="Base64";
String en="解密后";
System.out.println("原文:"+de);
date=Base64.encode(de.getBytes());
en=new String(date);
System.out.println("密文:"+en);
date=Base64.decode(en.getBytes());
de=new String(date);
System.out.println("解密:"+de);
System.out.println();

UrlBase64
UrlBase64用其他符号替代了url中不允许存在的符号 以用于在url中传递二进制数据 但可用于替代的符号实在有限故不同版本实现的替代符可能不同
org.bouncycastle.util.encoders.UrlBase64;

System.out.println("this is UrlBase64");
de="UrlBase64";
en="解密后";
System.out.println("原文:"+de);
date=UrlBase64.encode(de.getBytes());
en=new String(date);
System.out.println("密文:"+en);
date=UrlBase64.decode(en.getBytes());
de=new String(date);
System.out.println("解密:"+de);
System.out.println();
