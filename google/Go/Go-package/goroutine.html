goroutine 

goroutine 是一個類似線程的 可執行單元 (go 會負責 堆棧的 保存 恢復)
但她不由 os調度 而由 go runtime 調度(go 會使用全部 cpu 並行運行)
(goroutine 類似協程 但 其是並行運行的 而協程是 阻塞運行的)
(runtime.GOMAXPROCS(n) 可設置 最多使用cpu個數 默認使用 全部cpu)

goroutine 開銷 比 線程 地很多 可以 輕鬆的 創建 上百萬個

使用 go [函數調用]  創建一個 goroutine

example 

package main

import (
	"fmt"
	"time"
)

func main() {
	for i := 0; i < 10; i++ {
		go func(v int) {
			fmt.Println(v)
		}(i)
	}

	//等待其她 goroutine 結束
	time.Sleep(time.Second)
}

注意 

goroutine 並發調用的 函數 返回值 會被 忽略

main 自動運行在第一個 goroutine(main goroutine)上
main goroutine 結束 時 goruntine 會強制 結束掉其他 goroutine

channel 

go 使用 消息機制 而非 共享內存 在多個 goroutine間通信
channel 是go為此 提供的 解決方案

channel 是一個 類型安全的 雙向數據管道
使用 
make(chan,型別)
make(chan,型別,緩存長度)
定義
chan<-數據 寫入管道
[變量=]<-chan 讀取管道

example 

package main

import (
	"fmt"
)

func main() {
	chs := make([]chan int, 10)
	for i := 0; i < 10; i++ {
		chs[i] = make(chan int)

		go func(ch chan int, i int) {
			fmt.Println(i)
			ch <- i
		}(chs[i], i)
	}

	//等待其她 goroutine 結束
	sum := 0
	for _, ch := range chs {
		sum += <-ch
	}
	fmt.Println("sum =", sum)

	ch := make(chan int)
	go func() {
		for i := 0; i < 2; i++ {
			ch <- i + 100
		}
		close(ch)
	}()
	for i := range ch {
		fmt.Println(i)
	}
}

注意 

一般 channel 可以不用 close 
但如果 close 之後 不能再 傳輸 消息 
且 讀取消息的 < range 將返回

使用channel發送字符串 int等基本型別是安全的 
然發送指針 引用 不能保證數據安全 除非文檔有單獨說明如*regexp.Regexp
(因爲無法保證 指針 引用在發送後 接收前 其內容不被改變)


channel 的讀寫 都是 阻塞的 且一個不帶 緩衝的channel
一個寫入 一個 數據 就必須 等待 數據被讀出 才能再 寫入



channel 是 雙向的 但可以定義 一個 只能 讀/寫 的 channel
var name chan<-	//寫
var name <-chan	//讀

select 

select 類似 switch 但其 case 不是 條件語句
而是 channel 的 讀/寫 操作 一旦 某一操作 可執行 便 執行此 分支
(多個分支 同時滿足 條件 隨機執行其中一個 分支)
如果 都不可執行 等待 如果存在default 執行 default

example 

package main

import (
	"fmt"
	"time"
)

func main() {
	ch := make(chan int)
	for {
		go func(ch chan int) {
			select {
			case ch <- 0:
			case ch <- 1:
			case rand := <-ch:
				fmt.Println("lase second rand number[0,1] is", rand)
			}
		}(ch)
		time.Sleep(time.Second)
	}
}

channel 超時 

channel 沒有提供 超時 功能 
然 使用 select 可以 方便的 實現 超時
(既創建一個 超時 channel)

example 

package main

import (
	"fmt"
	"math/rand"
	"time"
)

func main() {
	ch := make(chan int)
	timeout := make(chan bool)

	go func(ch chan int) {
		for {
			ch <- rand.Intn(10)
			time.Sleep(time.Millisecond * 1500)
		}
	}(ch)

	for {
		go func(ch chan<- bool) {
			ch <- true
		}(timeout)

		select {
		case r := <-ch:
			fmt.Println("rand =", r)
		case <-timeout:
			fmt.Println("timeout deal")
		}
	}

}

sync 

channel 雖然 解決了一些 數據共享問題

然 依然 無法 避免 共享 內存
sync 包 提供 了 Mutex RWMutex ... 用以 處理 同步

sync.Once 

sync.Once 型別的 Do方法 用於 在多個goroutine中 只執行一次

package main

import (
	"fmt"
	"sync"
)

var once sync.Once

func work(i int, ch chan int32) {
	once.Do(func() {
		fmt.Println("only once")
	})

	fmt.Println(i)
	ch <- 0
}
func main() {
	count := 5
	ch := make(chan int32, 4)
	for i := 0; i < count; i++ {
		go work(i, ch)
	}

	for count > 0 {
		<-ch
		count -= 1
	}
}

sync.Mutex 

sync.Mutex 型別是一個 互斥量 
//鎖定/解鎖
func (m *Mutex) Lock()
func (m *Mutex) Unlock()

package main

import (
	"fmt"
	"sync"
	"time"
)

var mutex sync.Mutex

func work(ch chan int) {
	fmt.Println("work")

	mutex.Lock()
	fmt.Println("work ...")
	mutex.Unlock()

	ch <- 0
}
func main() {
	fmt.Println("Lock")
	mutex.Lock()
	fmt.Println("start goroutine")
	ch := make(chan int)
	go work(ch)
	time.Sleep(1000 * time.Millisecond)
	fmt.Println("Unlock")
	mutex.Unlock()

	<-ch
}
