掃描 搜索字符串 代碼混淆 PE文件格式 鏈接

掃描 

一般 對一個未知 程序的第一步 是上傳到 類似 
https://www.virustotal.com/
的網站 可以讓 多個 反病毒 軟體 進行掃描 看是否有 能 識別出的信息

之後可以 試下 google 文件的 hash 看是否有 識別出此 文件的 信息

搜索字符串 

搜索 程序中的 字符串 是一個簡單的 猜測程序功能的 方法
可以使用strings完成此步驟http://technet.microsoft.com/en-us/sysinternals/bb897439

代碼混淆 

代碼混淆 是用來隱藏 代碼功能的 一種技術 最常用的 是加壳

一般 壳會 壓縮 加密 程序 程序運行時先運行壳的脫壳代碼 之後運行程序
壳會嚴重 影響 靜態分析

一般可以使用 peid 檢測常用的 壳

注意 

peid 的一些插件 會在沒有任何提示的 情況下 運行 可執行文件
故 一定要在 安全的環境下 進行逆向

一般壳至少會 包含 LoadLibrary GetProcAddress 函數

PE文件格式 

windows 下的可執行文件 使用PE文件格式
其中包含了 windows 加載器 必要的代碼信息
(一些惡意代碼 會故意使用 過時的 兼容格式)

鏈接 

靜態鏈接很難被 識別 因為其不會在 PE 文件頭 中 存在任何信息

運行時鏈接 大量使用在 惡意代碼中
(LoadLibrary GetProcAddress LdrGetProcAddress LdrLoadDll)

動態鏈接 導入表中 記錄了動態鏈接的 詳細函數 信息
可以使用 Dependency Walker 查看http://www.dependencywalker.com/
