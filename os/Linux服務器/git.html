git 

git 分佈式版本控制系統 由GitHub的	Chris Wanstrath	PJ Hyett	Tom Preston-Werner	開發

然一般 會建立 一個用戶 作為 中央服務器 以方便 多人在此更新 代碼

https://github.com/

基本命令 

//初始化當前目錄 為git倉庫 
git init

//將 文件 提交到 緩存
git add FileName 

//將 緩存 提交到 倉庫
git commit -m "note text"

//查看 當前 倉庫 狀態
git status

//比較 文件變化情況
git diff FileName

//顯示 倉庫上傳 記錄 
git log [--pretty=oneline]
	--pretty=oneline 不顯示日期 作者

//到達指定 版本位置
git reset --hard Id
	HEAD 表示當前版本 HEAD^上個版本 HEAD~n上n個版本/HEAD^^...

//撤銷工作區的 所有修改
git checkout -- FileName

//撤銷 緩衝區的 修改 到當前版本
git reset HEAD FileName

//刪除 文件 (只是記錄到 緩衝區 還需要 git commit)
git rm FileName

分支 

//查看 分支 當前分支前 帶有 *
git branch [-r]
	-r 查看遠程分支

//創建 分支
git branch [dev]

//刪除 分支
git branch -d [dev]

//刪除 遠程 分支
git push origin :[dev]

//切換分支
git checkout [dev]

//創建 分支 並且 切換到此分支
git checkout -b [dev]

//合併指定分支到當前 分支
git merge [dev]


//建立分支 並且 更新為遠程指定分支
git checkout -b [dev] origin/[dev]

遠程倉庫 

//創建一個 裸倉庫(此類型 才可以被 推送)
git init --bare [name.git]

//下載 遠程 倉庫
git clone git@github.com:michaelliao/gitskills.git

//將指定分支 推送到 遠程 倉庫
git push origin [master]

//獲取 遠程倉庫中 指定 分支
git pull origin [master]

注意 

git默認 拒絕push操作
需要vi .git/config 添加
[receive]
denyCurrentBranch = ignore
